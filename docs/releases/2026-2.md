# CANFAR Science Platform Release 2026.2 - May 21, 2026

!!! success "CanfarSP 2026.2 - May 21, 2026"

    ### ✨ Features
    - **Science Portal** — The Science Portal user interface has been rewritten in [**Next.js**](https://nextjs.org/) with a refreshed interface. This release ships a **new `science-portal` Helm chart**, which is **not backward-compatible** with the previous chart: existing values files and upgrade paths from earlier releases do not carry over. Plan the upgrade using the current [Deployment Guide](https://www.opencadc.org/deployments/helm/science-platform/deployment/) and the new chart’s values schema.

    ### 📦 Deployment Notes
    - **[Deployment Guide](https://www.opencadc.org/deployments/helm/science-platform/deployment/)** — Helm-based deployment of the CANFAR Science Platform on Kubernetes: prerequisites (cluster, IVOA Service Registry), official chart repositories, and the recommended install order for POSIX mapper, Cavern, optional Kueue, Skaha, Science Portal, and Storage UI.
    - **`base` Helm chart retired** — The `base` chart is obsolete and must not be installed for deployments aligned with this release. Remove any automation that installs `science-platform/base` (including historical patterns that applied it in the `traefik` namespace); rely on the supported charts only.
    - **RBAC and namespaces** — RBAC and `Namespace` objects that were previously installed by `base` are now managed by the **`skaha`** chart, where they belong functionally.
    - **Ingress / Traefik outside this chart set** — Workloads that used to depend on `base` for Traefik must install an ingress stack separately. For example, deploy Traefik with the upstream [Traefik documentation](https://doc.traefik.io/traefik/getting-started/install-traefik/) (including the official Helm approach if that fits your cluster), or use your site’s equivalent ingress controller, according to your TLS and routing requirements—before or as you roll out the Science Platform charts.
