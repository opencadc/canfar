# CANFAR Science Platform Release 2026.2 - June 2, 2026

!!! success "CanfarSP 2026.2 - June 2, 2026"

    ### ✨ Features
    - New Science Portal, with Improved mobile device experience
    - [CARTA 5.1.0](https://github.com/CARTAvis/carta/releases/tag/v5.1.0) available
    
    ### 🐛 Fixes
    - Enabled the deletion of non-empty user storage folders through the Storage UI

    ### ⚙️ Technical Changes
    - All UI/UX components are now based on the same technology stack: Next.js, TypeScript, and React.  This will simplify and expediate modifications to the portal components in the future.
    - Addition of fsGroupChangePolicy - Ensures that kubernetes will avoid performing a recursive permission modification on job startup (although Franz says this isn't acutally working in their cluser??)
    - Readiness checks added for all session types.  Sessions aren't put in the 'Running' state until session initialization procedures are complete.
    - CARTA 5.1.0 with [psrecord](https://github.com/astrofrog/psrecord) available (Requires CanfarSP 2026.2)

    ### 📦 Deployment Notes
    - Improved and simplified deployment documentation
        - Complexity of matching keys between skaha and cavern no longer required
        - Link from the release notes 
        - Improved Kueue deployment documentation
    - The CANFAR Portal Branding Logo now configurable
    - Deployers have the option to configure skaha and cavern use the SRCNet Permisions API for authorization decisions.

    For full deployment instructions, please refer to the OpenCADc deployment repository:  https://github.com/opencadc/deployments
