# CANFAR Science Platform Release 2026.2 - May 21, 2026

!!! success "CanfarSP 2026.2 - May 21, 2026"

    ### ✨ Features
    - **Science Portal** — The Science Portal user interface has been rewritten in [**Next.js**](https://nextjs.org/) with a refreshed interface. This release ships a **new `science-portal` Helm chart**, which is **not backward-compatible** with the previous chart: existing values files and upgrade paths from earlier releases do not carry over. Plan the upgrade using the current [Deployment Guide](https://www.opencadc.org/deployments/helm/science-platform/deployment/) and the new chart’s values schema.
      - See the new Helm Chart at https://github.com/opencadc/science-portal/tree/main/helm
      - Details and documentation available at https://github.com/openacdc/science-portal.git
    - **Cavern** - Now enables the ability for new users to "self-allocate".  Permissions are controlled either by Group setting, or using the SRCNet Permissions API.
      - See the new Helm Chart configuration for allocation authorization: https://github.com/opencadc/deployments/blob/main/helm/applications/cavern/values.yaml#L89
    - The **Skaha** service is enjoying some optimizations that should help speed up Session creation
      - Now relies on the Cavern self-allocation for new users.
      - Small internal optimizations where effort was previously wasted.

    ### 🐛 Fixes
    - Removed hard-coded `fsGroup` setting on security context for Pods.  This causes issues when mounting multi-user volumes.

    ### 📦 Deployment Notes
    - **[Deployment Guide](https://www.opencadc.org/deployments/helm/science-platform/deployment/)** — Helm-based deployment of the CANFAR Science Platform on Kubernetes: prerequisites (cluster, IVOA Service Registry), official chart repositories, and the recommended install order for POSIX mapper, Cavern, optional Kueue, Skaha, Science Portal, and Storage UI.

    #### ⚠️ Potential Breaking Changes
    The **skaha** service is the only one that requires a Service Account as it will interact with the Kubernetes Cluster.
    - **[Skaha]** - `serviceAccountName` will need to be converted to the new `serviceAccount` object:
    ```yaml
    # Old
    #
    deployment:
      skaha:
        serviceAccountName: skaha

    # New
    # 
    serviceAccount:
      name: skaha
      create: false
    ```
